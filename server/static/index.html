What this is

RouteDiff treats the Traveling Salesperson Problem as a denoising task. Rather than building a tour step by step with a heuristic, it learns to begin from a graph of pure noise and gradually remove that noise until a short, valid round trip through every city emerges. It is the idea behind image diffusion models, applied instead to a discrete combinatorial object: the set of edges that form a tour.

The forward noising process

Training starts from a known good tour for each instance, produced by a classical solver. A forward process then corrupts that tour over a fixed number of timesteps using a discrete (D3PM-style) noise kernel: at each step every edge has some chance of being resampled at random, so a clean tour of N edges decays toward an essentially random edge set. By the final timestep almost no trace of the original route remains. This produces a ladder of progressively noisier versions of each tour for the model to learn from.

The GNN denoiser

A graph neural network learns to undo one step of that corruption. Given a noisy edge set, the city coordinates, and the current timestep, it predicts for every possible edge the probability that the edge belongs to the clean tour. Anisotropic gated graph convolutions let each edge's prediction depend on the whole graph, and a sinusoidal timestep embedding tells the network how much noise to expect. The model is deliberately small enough to train on a CPU.

Sampling a tour from noise

To solve a new instance the model runs the process in reverse. It starts from a random edge set and, from the last timestep down to the first, repeatedly predicts the clean tour and takes a small denoising step toward it. After the final step a greedy decoder turns the per-edge probabilities into a single valid Hamiltonian cycle, adding the most confident edges while rejecting any that would break the tour. The animation on the demo page shows exactly these intermediate states.
